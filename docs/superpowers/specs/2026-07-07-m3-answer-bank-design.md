# Ball De Fato — M3: Банк ответов ×100 (2000/язык)

**Дата:** 2026-07-07
**Статус:** утверждён
**Милстоун:** M3 (после M2)

## Цель

Увеличить число ответов шара с 20 до **2000 на каждый из 5 языков** (en/ru/de/es/fr),
нативно на язык, тональность **2:1:1** (1000 позитив / 500 нейтрал / 500 негатив).
Генерация — многоагентным workflow (согласие пользователя получено). Хранение — в
assets; логика тряски/UI не меняется.

## Решения (утверждены)

| Вопрос | Решение |
|---|---|
| Объём | 2000/язык, тональность 2:1:1 (1000/500/500) |
| Хранение | `assets/answers/<lang>.txt`, 1 ответ на строку |
| Стратегия | Нативно на каждый язык (не перевод) |
| Генерация | Многоагентный Workflow (fan-out + дедуп + loop-until-target) |
| Длина | Короткие предсказания, ≤ ~50 символов |

## Рантайм

- **`AnswerRepository`** (новый, `ai.mifmax.balldefato.logic`):
  - `fun load(context: Context, language: String): List<String>`.
  - Читает `assets/answers/<language>.txt` (UTF-8, по строке на ответ, пустые
    строки и строки, начинающиеся с `#`, игнорируются), дедупит (нормализация
    регистра/пробелов), возвращает список.
  - Фолбэк-цепочка: `<language>` → `en` → встроенный `R.array.responses` (20).
- **`MainActivity`**: определяет текущий язык (`resources.configuration.locales[0].language`),
  строит `AnswerPicker(AnswerRepository.load(this, lang))`.
- **`AnswerPicker`** — без изменений (список + RNG).
- Массив `responses` (20) в `strings.xml` остаётся аварийным фолбэком.

## Данные

- 5 файлов `assets/answers/{en,ru,de,es,fr}.txt`, каждый **2000** строк.
- Тональность 2:1:1 обеспечивается на генерации; в файле ответы перемешаны (picker
  берёт равномерно, порядок не важен).
- Каждый ответ: короткое мистическое предсказание в стиле Magic 8-Ball, ≤ ~50
  символов, без переносов строк (`\n` не используется — autosize подберёт размер).

## Генерация (Workflow)

- 15 потоков = 5 языков × 3 тональности, параллельно (cap concurrency).
- На поток: батчами агент выдаёт ~120 уникальных коротких ответов заданной
  тональности на заданном языке; в промпт передаётся случайный сэмпл уже собранных
  (чтобы снизить дубли). Аккумуляция с **дедупом** (нормализованный ключ) до целевого
  количества (1000/500/500), с потолком раундов (лог недобора — без тихого усечения).
- Фильтры: длина ≤ лимита, не пусто, соответствие тональности/языку, уникальность.
- Workflow возвращает структуру `{lang → [2000 ответов]}`; основной цикл записывает
  их в assets-файлы (скрипт workflow не пишет файлы сам).

## Тесты

- `AnswerRepositoryTest` (Robolectric): для каждого языка `load(...)` возвращает
  **2000** непустых **уникальных** ответов; отсутствующий язык падает на `en`;
  полностью отсутствующие assets → встроенные 20.
- Существующие 14 тестов остаются зелёными.

## Критерии готовности

1. 5 файлов `assets/answers/*.txt` по 2000 уникальных ответов (или зафиксированный
   лог недобора, если тон не добрал).
2. `AnswerRepository` + wiring в `MainActivity`; `AnswerRepositoryTest` зелёный.
3. `./gradlew testDebugUnitTest` и `assembleDebug` — зелёные; на устройстве шар при
   тряске выдаёт ответы из большого пула на языке приложения.

## Вне scope

- Смешивание языков в одном пуле; генерация на лету; изменение UI/логики тряски;
  идеальная «смысловая» уникальность (при ×100 контент вариативно-синонимичен).

## Реалистичная оговорка

2000 уникальных коротких предсказаний на язык = много перефразировок; дедуп убирает
точные повторы, но пул синонимичен по смыслу. Это следствие масштаба ×100.
